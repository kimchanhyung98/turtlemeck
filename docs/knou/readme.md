## 참가신청서 작성 정보

신청서에는 다음 정보가 필요하다.

- 작품명과 소속 지역
- 발표자와 팀원의 학번, 학년, 성명, 수신 가능한 이메일, 연락 가능한 전화번호
- 출품 분야
- 운영체제와 사용 언어
- 작품의 간단한 설명과 특징
- 컴퓨터과학과 수업 내용을 응용한 경우 해당 내용
- 사용한 오픈소스와 출처, 라이선스 정보
- 신청일과 참가 신청인 서명 또는 날인

양식에는 발표자 1명과 팀원 3명을 적는 칸이 있다. 공지에는 팀 최대 인원이 명시되어 있지 않으므로 5명 이상이라면 학과 사무실에 확인해야 한다.

소속 지역과 참가자 정보, 실제로 응용한 교과목은 참가자가 직접 작성해야 한다. 교과목은 가산점 대상이므로 실제 수업 내용과 작품의 연결 관계를 구체적으로 설명한다.

### 오픈소스와 라이선스

참가신청서의 출처 항목에는 최소한 다음 모델과 라이선스를 명시해야 한다.

- Apple Core ML Depth Anything V2 Small: Apache-2.0
- Apple PoseNet MobileNet 0.75 모델: Apache-2.0
- Apple PoseNet 샘플 코드를 사용한 부분: MIT
- turtlemeck 자체 소스: MIT

정확한 출처 URL과 고지 내용은 `Resources/ThirdPartyNotices.md`, `Resources/Apache-2.0.txt`, 루트 `LICENSE`를 기준으로 작성한다. 저장소 전체를 제출한다면 앱에 포함되지 않는 개발 도구도 구분해 설명한다.

### 실행 환경

공지에는 심사에 사용할 운영체제와 장비가 명시되어 있지 않다. turtlemeck는 macOS 전용이며 카메라와 Core ML 모델을 사용하므로 다음 사항을 제출 전에 확인해야 한다.

- 심사 환경에서 macOS 앱을 실행할 수 있는지 학과 사무실에 확인
- 지원 macOS 버전과 Apple Silicon·Intel 지원 여부를 실행 방법에 명시
- 최초 실행 시 필요한 카메라 권한 절차를 설명
- 제출한 앱 번들에서 PoseNet과 Depth Anything V2 모델이 정상적으로 로드되는지 확인
- 현재 패키지는 ad-hoc 서명 상태이며 Apple Developer ID 공증 대상이 아니므로 심사 장비에서 허용되는 실행 방법을 확인
- 별도 개발 도구 없이 실행할 수 있는 앱, ZIP 또는 DMG 제공
- 실행이 어려운 환경에서도 전체 기능을 확인할 수 있도록 시연 영상에 최초 실행, 보정, 자세 점검, 알림 과정을 포함

## 접수 절차

1. 참가 자격과 개인 또는 팀 참가 여부를 확인한다.
2. 첨부된 참가신청서를 작성하고 서명 또는 날인한다.
3. 앱, 실행 방법, 소스 파일, 발표 자료와 시연 영상을 준비한다.
4. 오픈소스 사용 내역과 라이선스 표기가 실제 제출물과 일치하는지 확인한다.
5. 앱을 새 환경에서 실행하고 카메라 권한, 모델 로드와 주요 흐름을 점검한다.
6. 2026년 7월 24일까지 신청서와 제출물을 이메일로 보낸다.
7. 대용량 파일을 Google Drive로 제출할 때는 심사 기간 동안 다운로드할 수 있도록 링크 권한을 확인한다.
8. 현장 행사에 참석할 계획이면 후속 공지의 참가비 입금과 참석자 등록을 별도로 진행한다. 선착순이므로 심사 결과 발표까지 기다리지 않는다.
9. 접수 확인 메일을 기다린다. 접수 후 주말과 공휴일을 제외한 2주 이내에 확인 메일을 받지 못하면 학과 사무실에 문의한다.

소스코드는 공개가 원칙이다. 공개 범위를 제한해야 한다면 제출 전에 학과와 협의할 수 있다. `.git`, `.build`, `debug`, `node_modules`와 개인 정보·임시 파일은 제출본에서 제외한다.

## 발표와 행사 참석

대상, 금상, 은상, 동상 후보자는 2026년 8월 29일 작품 발표에 참석해야 한다. 작품 설명은 8분, 질의응답은 5분이며 시간을 넘기면 발표가 중단된다. 당일 불참하면 심사에서 제외된다. 장려상은 작품 발표 없이 시상만 진행할 예정이다.

작품 전시는 온라인으로 진행되며 링크는 추후 공지된다. 최초 공지에서는 전체 참가 작품을 전시 대상으로 안내했지만 변경될 수 있다.

발표와 시상 행사는 대전 KT인재개발원 제1연수관 소강당에서 진행된다. 후속 공지에 따라 참가비를 입금한 뒤 참석자 등록 폼을 제출해야 하며, 신청은 선착순으로 마감될 수 있다. 8월 29일 소프트웨어경진대회는 15시에 시작하므로 발표자는 늦어도 시작 30분 전까지 발표 자료와 영상을 점검해야 한다.

8월 30일 시상식도 불참하면 시상에서 제외된다. 식사와 숙소는 학과에서 제공하며 세부 일정은 변경될 수 있다. 작품 설명 동영상의 별도 규격은 참가자에게 추후 안내된다.

2명 이상인 팀도 상장과 부상은 대표자 1명에게만 교부된다. 상금과 상품은 학과 운영 상황에 따라 바뀔 수 있다. 총장상을 제외한 학과 차원의 상장은 교내 포상 내역에 등록되지 않는다.

## 유의사항

- 타인의 저작권을 침해하거나 도용한 작품은 제출할 수 없다.
- 회사가 소유하고 상업적으로 사용하는 작품은 제출할 수 없다.
- 출품작의 저작권은 참가자 또는 참가팀에 있다.
- 학과는 출품작을 홍보하거나 전시하기 위한 비영리적 사용권을 가질 수 있다.
- 공개소스의 사용 내역과 라이선스를 참가신청서에 명확히 적고 해당 조건을 지켜야 한다.
