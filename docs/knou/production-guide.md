# 제출물 제작 실행 가이드

이 문서는 [제출 안내](submission.md)의 각 제출물을 실제로 만드는 도구와 절차를 정리한다. 2026년 7월 웹 검색으로 공식 문서 위주로 확인한 내용이며, 요금과 화면 구성은 바뀔 수 있다. macOS 15 Sequoia 이상, Apple Silicon 기준으로 작성했다.

## 권장 경로 요약

| 제출물 | 권장 도구 | 비용 |
|---|---|---|
| 참가신청서 | 작성된 `turtlemeck-application.docx` + Word for Mac | 무료 |
| 신청서 PDF | Word `파일 > 다른 이름으로 저장 > PDF`, 서명은 macOS 미리보기 | 무료 |
| 발표 자료 PPTX | 초안 `turtlemeck-presentation.pptx` + PowerPoint for Mac | 초안 무료 |
| 화면 녹화 | macOS 내장 `Cmd+Shift+5`, 자동 줌 연출은 Screen Studio, 무료 대안 QuickRecorder·OBS | 무료(Screen Studio만 유료) |
| 영상 편집·내보내기 | iMovie | 무료 |
| 내레이션 | iMovie 보이스오버 직접 녹음 | 무료 |
| 실행 안내 PDF | VS Code Markdown PDF 확장 또는 크롬 인쇄 | 무료 |
| 다이어그램 이미지 | mermaid.live 또는 mermaid-cli | 무료 |
| 대용량 전송 | Google Drive 링크 + 소용량 이메일 첨부 | 무료 |

## 1. 참가신청서 작성

참가신청서는 `docs/knou/generate-application.py`로 생성한다. 개인정보는 커밋되지 않는 `docs/knou/participants.env`에 두며, 항목은 `participants.env.example`을 복사해 채운다. `python3 -m pip install python-docx` 후 `python3 docs/knou/generate-application.py`를 실행하면 공식 HWP 양식(`application-form.hwp`)과 같은 표 구조·치수의 `turtlemeck-application.docx`와 `turtlemeck-application.pdf`(Chrome 필요)가 만들어진다. 생성된 docx는 Word for Mac에서 열어 확인·수정할 수 있다. HWP 형식 제출이 꼭 필요할 때만 아래 한컴독스 절차를 사용한다.

Mac에는 한글이 기본 설치되어 있지 않으므로 웹 기반 한컴독스를 사용한다. 무료 요금제로 hwp 편집과 hwp·PDF 다운로드가 가능하다.

### 한컴독스로 작성

1. `hancomdocs.com` 접속 후 `무료로 시작하기`로 가입한다. 구글·네이버·카카오 간편 가입이 가능하고 카드 등록은 필요 없다.
2. 홈 화면의 `업로드` 버튼을 누르거나 참가신청서 hwp 파일을 브라우저 창으로 드래그한다.
3. 업로드된 파일을 클릭하면 브라우저 안에서 한컴오피스 Web 편집기가 열린다. 표의 빈칸을 클릭해 작품명, 학번, 성명, 연락처 등을 입력한다.
4. hwp 저장: 편집기 상단 `파일 > 다운로드`에서 HWP 형식을 선택한다.
5. PDF 저장: 같은 `파일 > 다운로드` 메뉴에서 PDF를 선택한다. 별도 변환 도구가 필요 없다.
6. 다운로드한 hwp와 PDF를 다시 열어 표 밀림과 글꼴 깨짐을 확인한다. 웹 편집기와 데스크톱 한글의 렌더링이 미세하게 다를 수 있다.

무료 요금제는 저장공간 2GB, 파일당 1GB, 비영리 개인 용도 조건이다. 일부 자료에 무료 요금제의 다운로드 제한 언급이 있으므로 가입 직후 테스트 파일로 HWP 다운로드가 되는지 먼저 확인한다. 신청서가 편집 제한이 걸린 배포용 문서라면 어떤 도구로도 편집할 수 없으므로 학과에 편집 가능본을 요청한다.

### 대안

- 한컴오피스 한글 for Mac: 가정 및 학생용 다운로드형 99,000원(비상업용 1PC), 한컴독스 개인용 구독(월 6,900원, 연 49,000원)에도 설치형이 포함된다. Mac용 체험판은 공식 확인이 되지 않으므로 체험판을 전제로 계획하지 않는다.
- 폴라리스 오피스: 공식 고객지원 문서 기준 Mac에서 HWP 편집은 유료다. 가격 이점이 없어 권장하지 않는다.

### 서명 또는 도장 넣기

- HWP 단계: 편집기에서 도장 위치를 클릭하고 `입력 > 그림`으로 배경이 투명한 PNG 도장 이미지를 넣는다. 글과 겹치면 그림 속성에서 배치를 `글 앞으로`로 바꾼다.
- PDF 단계: PDF를 미리보기로 열고 마크업 도구막대의 서명 버튼에서 `서명 생성`을 누른다. 트랙패드에 손가락으로 서명하거나, 흰 종이에 한 서명을 카메라에 비춰 캡처한다. 삽입된 서명을 서명란으로 드래그하고 `⌘S`로 저장한다.
- 도장 사진의 흰 배경 제거: 미리보기에서 도장 사진을 열고 마크업의 인스턴트 알파(요술봉 아이콘)로 흰 배경을 클릭·드래그해 삭제한 뒤 PNG로 내보낸다. 해상도는 300dpi 이상이어야 흐릿하지 않다.

## 2. 발표 자료 PPTX

### 초안 파일

[발표 자료](presentation.md)의 10장 구성과 발표 대본을 옮긴 편집용 초안이 `turtlemeck-presentation.pptx`로 준비되어 있다.

- 슬라이드마다 발표 대본과 시간 배분이 발표자 노트에 들어 있다.
- 점선 상자는 스크린샷 자리표시자다. 최종 앱 화면을 캡처해 교체한다.
- 4·5·9번 슬라이드의 흐름도는 도형으로 그려져 있어 자유롭게 수정할 수 있다.
- 글꼴은 Pretendard로 지정되어 있다. 설치하지 않으면 대체 글꼴로 표시되므로 아래 글꼴 설치를 먼저 진행한다.

### Pretendard 글꼴 설치

Pretendard는 SIL OFL 라이선스라 상업적 사용, 수정, 재배포가 모두 가능하다.

1. 터미널에서 `brew install --cask font-pretendard`를 실행한다. Homebrew가 없으면 `github.com/orioncactus/pretendard`의 Releases에서 zip을 받아 압축을 풀고, otf 파일들을 서체 관리자(Font Book)로 드래그한다.
2. 발표 자료에는 가변 폰트(PretendardVariable.ttf)가 아니라 정적 otf(Regular, Bold 등)를 사용한다. 가변 폰트는 PowerPoint 호환성이 떨어질 수 있다.
3. 설치 후 PowerPoint나 Keynote를 재실행해야 글꼴 목록에 나타난다.

### PowerPoint for Mac에서 마무리 (작성자 환경에 설치됨)

1. `turtlemeck-presentation.pptx`를 열어 자리표시자를 스크린샷으로 교체하고 내용을 다듬는다.
2. 글꼴 포함 저장: 메뉴 막대 `PowerPoint > 환경설정 > 출력 및 공유 그룹의 저장`에서 `파일에 글꼴 포함`을 체크하고 저장한다. 발표만 할 것이므로 `프레젠테이션에 사용된 문자만 포함`으로 충분하다. 이렇게 하면 Windows 발표 PC에 Pretendard가 없어도 깨지지 않는다.
3. 글꼴 포함은 Microsoft 365와 2021 이후 버전에서만 동작한다. PowerPoint 2016 for Mac은 불가하다.

### 무료 대안

- Keynote: 작성 후 `파일 > 내보내기 > PowerPoint`로 pptx를 만든다. 매직 무브 같은 Keynote 고유 효과는 변환 시 단순화되거나 사라지므로 전환은 디졸브·밀기 같은 기본형만 쓴다. Keynote는 글꼴 포함 기능이 없다.
- Google Slides: `slides.google.com`에서 작성 후 `파일 > 다운로드 > Microsoft PowerPoint(.pptx)`로 받는다. 글꼴 포함은 되지 않는다.
- PowerPoint 웹 버전: 무료 Microsoft 계정으로 `office.com`에서 편집할 수 있지만 글꼴 포함 저장이 없다. Microsoft 365 Personal 구독은 월 12,500원이다.

### Windows 발표 PC 대비

- 전환 효과는 기본형만 사용한다.
- `파일 > 내보내기 > PDF`(Keynote) 또는 PDF 저장으로 백업본을 만들어 USB와 드라이브에 함께 넣는다. PDF는 어떤 PC에서도 레이아웃이 유지된다.
- 사용한 글꼴 otf 파일을 `fonts` 폴더로 묶어 지참한다. OFL이므로 배포가 합법이고, 발표 PC에서 우클릭 설치 후 PowerPoint를 재시작하면 된다.
- 발표 당일 시작 30분 전 점검 시간에 실제 PC에서 슬라이드 쇼를 끝까지 돌려 보고, 문제가 있으면 즉시 PDF로 전환한다.

### 흐름도 이미지가 필요할 때

[발표 자료](presentation.md)의 mermaid 흐름도를 이미지로 쓰려면:

- mermaid.live: 접속해 왼쪽 Code 탭에 코드를 붙여넣고, 왼쪽 아래 Actions 패널에서 PNG size를 Width 3000~4000으로 지정한 뒤 PNG 버튼을 누른다. 5000을 넘기면 품질 저하나 빈 파일 버그가 보고되어 있다.
- 로컬 변환: 코드를 `flow.mmd`로 저장하고 `npx -y -p @mermaid-js/mermaid-cli mmdc -i flow.mmd -o flow.png -w 2400 -s 4 -b white`를 실행한다. 슬라이드 배경이 유색이면 `-b transparent`를 쓴다.

## 3. 시연 영상

[시연 영상](demo-video.md)의 장면 구성을 다음 절차로 촬영·편집한다.

### 촬영 환경 준비

1. 알림 차단: 제어 센터에서 집중 모드 섹션을 열고 `방해금지 모드 > 1시간 동안`을 선택한다.
2. 화면 확대 준비: `시스템 설정 > 손쉬운 사용 > 확대/축소`에서 `스크롤 제스처와 보조 키를 사용하여 확대/축소`를 켠다. 녹화 중 Control 키를 누른 채 두 손가락으로 위로 스크롤하면 포인터 중심으로 화면이 확대된다. 메뉴 막대 아이콘 클릭과 팝오버 조작 장면에 사용한다.
3. UI 크게 보이기: `시스템 설정 > 디스플레이`에서 해상도를 `텍스트 크게` 쪽으로 바꾸면 메뉴 막대와 앱 UI가 크게 렌더링된다. 촬영 후 되돌린다.
4. 메뉴 막대 정리: 불필요한 서드파티 아이콘은 앱을 종료하거나 Command+드래그로 제거한다. 녹화 중지 버튼 아이콘도 메뉴 막대에 표시되므로 시연할 앱 아이콘과 겹치지 않게 정리한다.
5. Dock 정리: `시스템 설정 > 데스크탑 및 Dock`에서 `자동으로 Dock 가리기`를 켠다.

### 녹화 도구 선택

기본 경로는 아래의 `Cmd+Shift+5`(무료·무워터마크)다. 메뉴 팝오버 확대를 자동으로 처리하거나 웹캠 합성을 녹화 단계에서 끝내고 싶으면 다음 도구를 쓴다. 가격과 유지보수 상태는 2026년 7월에 공식 사이트와 GitHub에서 확인한 값이다.

| 도구 | 비용 | 특징 |
|---|---|---|
| Screen Studio | 월 $20 구독(연 결제 시 월 $9) | 클릭 시 자동 줌, 커서 스무딩, 웹캠 오버레이. 소개 영상 완성도가 가장 높다. 미결제 상태로 녹화·편집을 무제한 연습할 수 있고 내보내기만 잠긴다(워터마크 방식이 아님). |
| QuickRecorder | 무료(오픈소스) | 경량 메뉴 막대 앱. 창·영역 단위 녹화, 마우스 강조, macOS 14+ Presenter Overlay로 웹캠 실시간 합성, 시스템 오디오 녹음. 자동 줌은 없다. |
| OBS Studio | 무료(오픈소스) | 장면(Scene)에 화면과 웹캠 소스를 배치해 PiP를 한 번의 녹화로 완성한다. 팝오버 크롭 확대 장면과 웹캠 자세 장면을 단축키로 전환할 수 있다. 초기 설정 학습이 필요하다. |

- Screen Studio: `screen.studio`에서 설치하고 화면 기록·카메라·마이크 권한을 허용한다. 화면/창/영역과 웹캠·오디오를 선택해 녹화하면 종료 시 자동 줌·커서 스무딩이 적용된 편집기가 열린다. 타임라인에서 줌 구간을 드래그로 조정하고 mp4(최대 4K 60fps)로 내보낸다. 2026년 5월(3.7.1)까지 갱신된 활성 프로젝트이며 구독 전용이다(일회성 판매는 2025년에 중단). 무료로 전 과정을 연습한 뒤 제출 직전 1개월만 결제하는 방식이 가능하다.
- QuickRecorder: `brew install lihaoyun6/tap/quickrecorder` 또는 GitHub Releases로 설치한다. 메뉴 막대 아이콘에서 화면/창/영역을 고르고 설정에서 마우스 강조와 카메라(Presenter Overlay)를 켠다. 녹화를 마치면 지정 폴더에 바로 mp4/mov가 저장된다. 마지막 릴리스가 2025년 6월(v1.6.9)로 유지보수가 느려졌고, macOS 15에서 화면 기록 권한 재확인 팝업이 주기적으로 뜨는 알려진 불편이 있다.
- OBS Studio: `obsproject.com` 또는 `brew install --cask obs`로 설치하고 자동 구성 마법사에서 녹화 최적화를 선택한다. `OBS Studio 메뉴 > Review App Permissions`에서 화면 기록·카메라·마이크를 허용하고 재시작한다. 장면에 `macOS 화면 캡처`와 `비디오 캡처 장치`(웹캠) 소스를 추가하고 웹캠을 구석으로 배치한 뒤, 설정의 비디오를 1920×1080으로 지정하고 테스트 녹화를 거쳐 본 녹화를 한다. mkv로 녹화됐다면 `파일 > 녹화 리먹스`로 mp4로 바꾼다. 2026년 4월(32.1.2)까지 갱신된 활성 프로젝트다.

한때 널리 쓰인 Kap은 2022년 10월 이후 릴리스가 멈췄고 macOS 15에서 화면 기록 권한 부여가 실패한다는 미해결 보고가 많아 제외한다. 자동 줌 계열에서 일회성 구매를 원하면 FocuSee Advanced Lifetime($199.99)이 대안이지만, 무료판 1회 내보내기의 워터마크 유무가 공식 문서 간에 상충하므로 제출용으로 쓰기 전에 직접 테스트한다.

### 웹캠(자세 장면) 촬영과 카메라 충돌

turtlemeck 자체가 내장 카메라를 사용하므로, 녹화 도구의 웹캠 오버레이가 같은 내장 카메라를 동시에 열면 충돌할 수 있다(이 프로젝트에서 라이브 앱과 외부 캡처 도구가 카메라를 두고 충돌한 실측 기록이 있다). 다음 순서로 회피한다.

1. 기본: 자세 장면을 별도 테이크로 촬영한다. QuickTime의 `파일 > 새로운 동영상 녹화`로 자세 변화만 찍고, iMovie에서 화면 녹화 클립 위 트랙에 올려 좌측 팝업 메뉴의 `픽처 인 픽처`로 합성한다.
2. 동시 촬영이 필요하면 iPhone 연속성 카메라(macOS 13+)를 녹화 도구의 웹캠 소스로 지정해 내장 카메라를 앱 전용으로 남긴다.
3. 내장 카메라 동시 접근을 꼭 쓰려면 본 촬영 전에 같은 구성으로 리허설해 충돌 여부를 확인한다.

앱 화면에는 카메라 영상이 표시되지 않으므로, 판정 과정은 팝오버의 상태 변화(정상 → 주의)로 보여주고 자세 변화 자체는 웹캠 장면으로 보여준다.

### 녹화 (Cmd+Shift+5)

1. `Shift+Command+5`로 화면 하단 도구 막대를 띄운다.
2. `옵션` 버튼에서 설정한다: 저장 위치, 타이머(준비 시간 필요 시), 마이크(내레이션을 따로 입힐 것이므로 `없음`), `마우스 클릭 표시` 체크(클릭 시 포인터 주위에 원이 표시됨).
3. `전체 화면 기록`을 선택하고 `기록`을 누른다. 메뉴 막대가 프레임에 들어가야 하므로 전체 화면 기록을 사용한다.
4. 시연을 진행하고, 메뉴 막대의 중지 버튼 또는 `Command+Control+Esc`로 종료한다.
5. 결과물은 `.mov`로 저장된다. iMovie에서 편집 후 mp4로 내보내므로 이 단계에서는 변환하지 않는다.
6. 같은 장면을 두 번 이상 촬영하고, 본 촬영 전에 10초 테스트 녹화로 화질과 확대 동작을 확인한다.

### 편집 (iMovie)

한국어 자동 자막이 필요하면 CapCut 데스크톱(기본 편집·1080p 내보내기 무료, 타임라인 끝의 CapCut 아웃트로 클립을 삭제하면 워터마크 없음)을 쓸 수 있다. 다만 약관상 콘텐츠에 광범위한 라이선스를 부여하므로 얼굴이 담긴 이 영상은 iMovie 편집을 기본으로 한다.

1. 프로젝트 생성: iMovie 실행 후 `새로 생성 > 동영상`을 선택한다(예고편 아님).
2. 가져오기: `파일 > 미디어 가져오기(⌘I)`로 mov 파일을 불러와 타임라인으로 드래그한다.
3. 자르기: 자를 지점에 재생헤드를 놓고 `⌘B`(수정 > 클립 분할)로 나눈 뒤 필요 없는 조각을 선택해 Delete로 지운다. `⌘B`는 타임라인에 포커스가 있을 때만 동작한다.
4. 속도 조절: 단조로운 구간은 클립 선택 후 뷰어 위 속도계 모양 버튼에서 `빠르게`를 골라 2~8배로 줄인다.
5. 자막: 브라우저 위 `타이틀` 탭에서 단순한 스타일(하단 자막은 Lower Third 계열, 장면 제목은 Standard)을 골라 타임라인의 클립 위에 드래그한다. 뷰어에서 텍스트를 더블클릭해 입력하고 폰트 컨트롤로 글꼴과 크기를 바꾼다. 표시 시간은 보라색 타이틀 바 양 끝을 드래그해 조절한다.
6. 화면 확대: 확대할 구간을 `⌘B`로 앞뒤 분할해 독립 클립으로 만들고, 뷰어 위 `크로핑` 버튼에서 고정 확대는 `크로프하여 채우기`, 서서히 줌인은 `Ken Burns`(시작 프레임 전체, 종료 프레임 확대 영역)를 적용한다.
7. 배경음악: 음원을 타임라인 아래 음악 영역으로 드래그하고, 파형 가운데 볼륨 라인을 20~30% 수준으로 내린다. 클립 가장자리의 페이드 핸들로 페이드 인·아웃을 만든다. 음원은 iMovie 내장 사운드나 무료 라이선스 음원만 사용한다.

MacBook 내장 화면은 16:10이고 iMovie 프로젝트는 16:9라 위아래가 잘리거나 여백이 생길 수 있다. `크로프하여 채우기`로 16:9 영역을 잡거나, 녹화 단계에서 선택 영역을 16:9에 가깝게 지정한다.

### 내레이션

영상을 먼저 완성한 뒤 iMovie 보이스오버로 [시연 영상](demo-video.md)의 내레이션을 입힌다. 화면 조작 소리가 섞이지 않도록 시연 화면과 음성은 분리해서 작업한다.

1. 재생헤드를 녹음 시작 지점에 놓고 뷰어 아래 마이크 모양 `보이스오버 녹음` 버튼을 누른다(단축키 V).
2. `보이스오버 옵션`에서 입력 소스를 선택하고, 말할 때 오디오 미터가 초록색을 유지하도록 음량을 조절한다. `프로젝트 음소거`를 켜면 재생음이 스피커로 새어 들어가지 않는다.
3. 빨간 버튼을 누르면 3초 카운트다운 후 시작된다. 문단 단위로 끊어 읽고, 틀린 문장은 그 클립만 지우고 다시 녹음한다.
4. 잡음 제거: 녹음 클립 선택 후 뷰어 위 `노이즈 감소 및 이퀄라이저`에서 `배경 노이즈 감소`를 30~60% 범위로 조절하고, 이퀄라이저를 `음성 품질 향상`으로 둔다. 100%에 가까우면 목소리가 뭉개진다.
5. 음량 보정: 오디오 클립 선택 후 `음량 > 자동`을 누른다.

녹음은 창문을 닫은 조용한 방에서, 에어컨과 팬을 끄고, 입과 마이크 거리를 20~30cm로 일정하게 유지한다. 녹음 중 키보드와 트랙패드를 조작하지 않는다.

AI 음성이 필요하면 네이버 클로바더빙 무료 요금제(월 15,000자, 다운로드 20회 — 서드파티 자료 기준이므로 로그인 후 실제 한도 확인)를 쓸 수 있다. 완성 영상을 업로드해 타이밍에 맞춰 더빙을 배치하고 음원만 받아 iMovie에서 합친다. 무료 요금제는 출처 표기가 필수이므로 영상 크레딧에 `내레이션: 네이버 클로바더빙` 문구를 넣어야 하며, 상금이 있는 대회 제출이 허용 범위인지 이용 정책을 직접 확인한다. 고유명사는 `터틀멕`처럼 한글로 입력해야 발음이 자연스럽다. 타입캐스트 무료 요금제는 상업적 이용 불가에 출처 표기 의무까지 있어 이점이 없다.

### 내보내기 (1080p mp4)

1. `파일 > 공유 > 파일`을 선택한다.
2. 해상도 `1080p (1920x1080)`, 품질 `높음`, 압축 `고품질`로 설정한다.
3. 품질에서 `최고(ProRes)`를 고르면 mov로 저장되고 파일이 4배 이상 커지므로 선택하지 않는다. `높음`이면 H.264 mp4가 생성된다.
4. 내보낸 파일의 확장자가 `.mp4`인지 확인하고, QuickTime Player로 열어 길이, 자막, 오디오 싱크를 확인한다. 2분 30초 1080p 기준 수백 MB가 될 수 있으므로 제출은 Google Drive 링크로 한다.

편집 없이 mov만 변환해야 할 때는 HandBrake(무료)에서 프리셋 `Fast 1080p30`을 쓰거나 `ffmpeg -i 입력.mov -c:v libx264 -crf 20 -c:a aac -movflags +faststart -vf scale=1920:-2 출력.mp4`를 실행한다.

## 4. 실행 안내 PDF

[실행 안내](run-guide.md)를 PDF로 변환해 `README.pdf`를 만든다.

### VS Code Markdown PDF 확장 (권장)

1. VS Code 확장 탭에서 `Markdown PDF`(yzane)를 설치한다.
2. `run-guide.md`를 열고 `⌘⇧P > Markdown PDF: Export (pdf)`를 실행한다. 같은 폴더에 PDF가 생성된다.
3. 한글은 시스템 폰트로 렌더링되어 자동으로 포함되므로 별도 설정이 필요 없다.
4. 생성된 PDF의 표가 페이지 경계에서 잘리지 않는지 전체 페이지를 확인한다.

### 대안: 크롬 인쇄

1. Markdown PDF 확장의 `Export (html)`로 HTML을 만들고 크롬으로 연다.
2. `⌘P`에서 대상을 `PDF로 저장`으로 바꾼다.
3. `설정 더보기`에서 `머리글 및 바닥글`을 해제하고 `배경 그래픽`을 체크한 뒤 저장한다.

pandoc과 LaTeX 조합은 설치와 폰트 설정에 시간이 들어 마감 직전에는 권장하지 않는다.

### 심사자 실행 안내에 반영할 내용

macOS 15 Sequoia부터 우클릭 후 열기로 Gatekeeper를 우회하는 방법이 제거되었다. 실행 안내에는 다음 순서를 정확히 적는다.

1. 앱을 더블클릭해 실행을 시도하면 차단 대화상자가 뜬다. `휴지통으로 이동`이 아니라 `완료`를 누른다.
2. `시스템 설정 > 개인정보 보호 및 보안`을 열고 아래 보안 섹션까지 스크롤하면 `그래도 열기` 버튼이 있다.
3. `그래도 열기`는 실행 시도 후 약 1시간 동안만 표시된다. 사라졌으면 앱을 다시 더블클릭해 대화상자를 다시 띄운다.
4. 확인 대화상자에서 다시 `그래도 열기`를 누르고 관리자 암호 또는 Touch ID로 승인한다. 승인은 한 번만 하면 된다.

터미널에 익숙한 심사자를 위한 보조 수단으로 `xattr -d com.apple.quarantine /Applications/turtlemeck.app`을 병기할 수 있다. 시스템 전체 Gatekeeper를 끄는 `spctl --master-disable`은 안내문에 넣지 않는다.

## 5. 제출 파일 전송

### Google Drive 업로드와 공유

1. `drive.google.com`에 개인 Gmail 계정으로 로그인한다. 학교·회사 Workspace 계정은 외부 공유가 막혀 `링크가 있는 모든 사용자` 옵션이 없을 수 있다.
2. `신규 > 파일 업로드`로 zip·dmg·mp4를 개별 파일로 올린다. 폴더 공유는 심사자가 내려받을 때 여러 zip으로 쪼개질 수 있어 권장하지 않는다.
3. 파일 우클릭 `공유 > 공유`에서 일반 액세스를 `제한됨`에서 `링크가 있는 모든 사용자`로 바꾸고 역할이 `뷰어`인지 확인한다.
4. 대화상자 우측 상단 톱니바퀴에서 `뷰어 및 댓글 작성자에게 다운로드, 인쇄, 복사 옵션 표시`가 켜져 있는지 확인한다.
5. `링크 복사`로 파일별 링크를 복사한다.
6. 시크릿 창(`Cmd+Shift+N`)에서 링크를 열어 로그인 없이 미리보기와 다운로드가 되는지, 받은 zip이 정상적으로 풀리는지 확인한다.

100MB가 넘는 파일은 다운로드 시 `바이러스를 검사할 수 없음` 경고가 뜬다. 정상 동작이므로 제출 이메일에 다음 안내를 넣는다.

```text
파일이 100MB를 초과하여 다운로드 시 Google Drive의 "바이러스를 검사할 수 없음" 안내가
표시될 수 있습니다. 파일 크기 때문에 나타나는 정상적인 안내이며,
[그래도 다운로드] 버튼을 누르시면 정상적으로 다운로드됩니다.
```

### 제출 후 파일 수정

심사 기간에 파일을 고쳐야 하면 삭제 후 재업로드하지 않는다. 새 파일은 다른 ID를 받아 제출한 링크가 깨진다. 대신 파일 선택 후 `더보기(⋮) > 버전 관리 > 새 버전 업로드`를 쓰면 링크가 유지된다. 이전 버전은 30일 후 삭제될 수 있으므로 필요하면 `영구 보관`을 켠다. 마감 이후 수정은 규정 위반이 될 수 있으니 먼저 학과에 문의한다.

### 이메일 첨부 구분

- 직접 첨부: 참가신청서 hwp·PDF, 실행 안내 PDF처럼 작은 파일.
- Drive 링크: 앱 zip·dmg, 소스 zip, 시연 영상, (용량이 크면) 발표 자료.

Gmail 발신 한도는 첨부 합계 25MB이고 수신 측 대학 메일 서버는 통상 10~25MB다. 첨부 합계가 10MB를 넘으면 반송될 수 있으므로 애매하면 링크로 보낸다. 발송 후 반송 메일(Mail Delivery Subsystem)이 오지 않았는지 확인한다.

## 확인하지 못한 사항

- 한컴독스 무료 요금제의 다운로드 제한 여부: 공식 문서에 명시가 없어 가입 직후 직접 테스트가 필요하다.
- 클로바더빙 무료 한도(월 15,000자, 다운로드 20회): 서드파티 자료 교차 확인 값이므로 로그인 후 실제 화면에서 확인한다.
- 상금이 있는 대회 제출이 클로바더빙·타입캐스트 무료 요금제의 비상업 범위에 드는지: 약관에서 직접 확인하지 못했다.
- 방송대 메일 서버(mail.knou.ac.kr)의 수신 첨부 한도: 확인하지 못했으므로 대용량은 예외 없이 링크로 보낸다.
- Mac PowerPoint 한국어판의 `글꼴 포함` 체크박스 정확한 표기: 영문판 공식 문서 기준이다.
