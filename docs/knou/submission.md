# 제출 안내

## 제출 파일 구성

다음 구조로 제출용 디렉토리를 만든다.

```text
turtlemeck-submission/
├── 01-application/
│   ├── turtlemeck-application.hwp
│   └── turtlemeck-application.pdf
├── 02-app/
│   ├── turtlemeck.dmg
│   ├── turtlemeck.zip
│   └── SHA256SUMS
├── 03-source/
│   └── turtlemeck-source.zip
├── 04-presentation/
│   └── turtlemeck-presentation.pptx
├── 05-video/
│   └── turtlemeck-demo.mp4
└── README.pdf
```

공식 신청서는 HWP 파일을 제출한다. PDF는 심사자가 내용을 바로 확인할 수 있도록 추가하며 HWP를 대체하지 않는다. `README.pdf`는 [실행 안내](run-guide.md)를 PDF로 변환한 파일이다.

## 소스 파일 준비

소스 압축에는 앱을 빌드하고 구조를 확인하는 데 필요한 다음 항목을 포함한다.

- `Sources`
- `Tests`
- `Resources`
- `docs`
- `Package.swift`
- `Makefile`
- `package.sh`
- `README.md`
- `LICENSE`
- 의존성 잠금 파일과 빌드·테스트 워크플로우

다음 항목은 제외한다.

- `.git`
- `.build`
- `debug`
- `node_modules`
- 운영체제 임시 파일
- 개인 정보, 계정 정보와 로컬 설정
- 촬영 과정에서 생성된 이미지와 분석 결과

Core ML 모델과 라이선스 고지 파일은 앱 실행과 출처 확인에 필요하므로 `Resources`에서 제외하지 않는다.

## 최종 산출물 확인

- 참가신청서에 신청일과 서명 또는 날인이 있음
- 참가자 연락처와 소속 지역이 정확함
- 실제 적용한 교과목과 작품의 연결 관계를 작성함
- 앱이 제출할 DMG와 ZIP에서 실행됨
- Apple Silicon과 Intel 실행 파일이 포함됨
- 카메라 권한, 기준 자세 보정과 자세 점검이 동작함
- 모델 파일과 제3자 라이선스 고지가 앱에 포함됨
- 실행 안내서와 시연 영상의 설명이 최종 앱 동작과 일치함
- 발표 자료에 개발 목적·배경, 내용·방법, 프로그램 구조, 실행 예와 결론이 포함됨
- Google Drive 링크를 로그인하지 않은 브라우저에서 내려받을 수 있음
- 제출 파일명과 체크섬이 최종 파일을 기준으로 생성됨

## 이메일 문안

제목:

```text
[소프트웨어경진대회 참가 신청_참가자명]
```

본문:

```text
안녕하세요.

2026 컴퓨터과학과 총장배 소프트웨어경진대회 참가 신청 자료를 제출합니다.

- 작품명: turtlemeck
- 출품 분야: 유틸리티 앱
- 참가 형태: [개인 또는 팀]
- 발표자: [성명]
- 연락처: [전화번호]
- 제출 파일: 참가신청서, 실행 가능한 앱, 실행 안내, 소스 파일, 발표 자료, 시연 영상
- 다운로드 링크: [Google Drive 링크]

첨부 및 링크에 문제가 있으면 위 연락처로 알려 주시기 바랍니다.
감사합니다.

[성명]
```

## 접수 후

1. 보낸 메일과 최종 제출 파일을 그대로 보관한다.
2. Google Drive 파일을 수정하거나 같은 이름으로 교체하지 않는다. 수정이 필요하면 학과에 먼저 문의한다.
3. 주말과 공휴일을 제외한 2주 이내에 접수 확인 메일이 오지 않으면 컴퓨터과학과 사무실에 문의한다.
4. 현장 참석자는 참가비 입금과 참석자 등록을 별도로 진행한다.
5. 1차 심사 결과와 작품 설명 동영상의 추가 규격을 확인한다.
6. 발표 대상이면 8분 발표와 5분 질의응답을 반복 연습한다.
