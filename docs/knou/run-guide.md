# turtlemeck 실행 안내

이 문서는 심사용 앱과 함께 제출한다.

## 작품 정보

| 항목 | 내용 |
|---|---|
| 작품명 | turtlemeck |
| 버전 | 1.0 |
| 지원 운영체제 | macOS 15.0 이상 |
| 지원 CPU | Apple Silicon, Intel |
| 필수 장치 | 카메라가 있는 Mac |
| 네트워크 | 필요 없음 |

## 설치

### DMG 사용

1. `turtlemeck.dmg`를 연다.
2. `turtlemeck.app`을 `Applications` 폴더로 복사한다.
3. `Applications`에서 앱을 실행한다.

### ZIP 사용

1. `turtlemeck.zip`의 압축을 푼다.
2. `turtlemeck.app`을 `Applications` 폴더로 옮긴다.
3. 앱을 실행한다.

현재 제출용 패키지는 ad-hoc 서명 상태이며 Apple Developer ID 공증을 받지 않았다. macOS가 실행을 차단하면 다음 순서로 허용한다.

1. 앱을 더블클릭해 실행을 시도하고, 차단 대화상자에서 `휴지통으로 이동`이 아니라 `완료`를 누른다.
2. `시스템 설정 > 개인정보 보호 및 보안`을 열고 아래 보안 섹션의 `그래도 열기`를 누른다. 이 버튼은 실행 시도 후 약 1시간 동안만 표시되며, 사라졌으면 앱을 다시 더블클릭해 대화상자를 다시 띄운다.
3. 확인 대화상자에서 다시 `그래도 열기`를 누르고 관리자 암호 또는 Touch ID로 승인한다. 승인은 최초 1회만 필요하다.

심사 환경의 보안 정책상 허용되지 않으면 시연 영상으로 전체 흐름을 확인한다.

## 최초 실행

1. 앱을 실행하면 Dock이 아니라 메뉴 막대에 거북이 아이콘이 나타난다.
2. 아이콘을 눌러 앱 화면을 연다.
3. macOS의 카메라 권한 요청에서 허용을 선택한다.
4. 바른 자세로 앉아 몸을 움직이지 않고 기준 자세 보정이 끝날 때까지 기다린다.
5. `기준 자세 저장됨`이 표시되면 정기 점검이 시작된다.

카메라 권한을 거부했다면 앱의 `카메라 권한 설정`을 눌러 `시스템 설정 > 개인정보 보호 및 보안 > 카메라`에서 turtlemeck를 허용한다.

## 주요 기능 확인

### 정상 자세

1. 기준 자세와 비슷한 위치에서 바르게 앉는다.
2. `확인`을 누른다.
3. 분석 가능한 입력이면 상태가 `자세: 정상`으로 표시된다.

### 주의 자세와 알림

1. 점검 주기를 15초로 설정한다.
2. `배너 알림`을 켜고 알림 권한을 허용한다.
3. 기준 자세보다 머리를 앞으로 이동한 상태를 두 번의 점검 동안 유지한다.
4. 연속된 악화가 확인되면 `자세: 주의`와 알림이 표시된다.

촬영 구도와 개인 기준에 따라 결과가 달라질 수 있다. 상체가 잘리거나 사람이 보이지 않고 입력이 불안정하면 앱은 나쁜 자세로 단정하지 않고 판단을 보류한다.

### 회복과 통계

1. 기준 자세와 비슷한 자세로 돌아온다.
2. 두 번의 점검 동안 자세를 유지한다.
3. 정상 상태로 회복되는지 확인한다.
4. `오늘 요약`에서 바른 자세 시간, 주의 자세 시간, 주의 전환, 회복과 알림 횟수를 확인한다.

### 기타 기능

- `확인`: 다음 예약을 기다리지 않고 점검
- `중지`·`시작`: 카메라 점검과 알림 일시정지 또는 재개
- `보정`: 현재의 바른 자세로 개인 기준 다시 생성
- `20분 스누즈`: 자세 점검은 유지하고 알림만 일시 중지
- `로그인 시 자동 실행`: macOS 로그인 시 앱 자동 실행
- `개인정보 · 비의료 안내`: 처리 범위와 의료기기가 아니라는 안내 확인

## 데이터 처리

- 자세 분석은 기기 안에서 실행된다.
- 운영 모드에서는 촬영 이미지를 저장하거나 외부로 전송하지 않는다.
- 개인 기준, 설정과 자세 통계는 로컬에 저장한다.
- 앱은 의료기기가 아니며 질병이나 통증을 진단하지 않는다.

## 문제 해결

| 증상 | 확인할 내용 |
|---|---|
| 메뉴 막대 아이콘이 보이지 않음 | 앱이 실행 중인지 확인하고 메뉴 막대의 가려진 아이콘 영역을 펼친다. |
| 카메라 권한 필요 | 앱의 `카메라 권한 설정`에서 권한을 허용한 뒤 앱을 다시 실행한다. |
| 보정 실패 | 얼굴과 양쪽 어깨가 화면에 들어오도록 앉고 조명을 확인한 뒤 `보정`을 다시 누른다. |
| 판단 보류가 계속됨 | 카메라 정면에 한 명만 앉고 상체가 잘리지 않도록 구도를 조정한다. |
| 알림이 오지 않음 | 배너 또는 소리 설정, macOS 알림 권한과 20분 스누즈 상태를 확인한다. |
| 앱이 열리지 않음 | macOS 버전과 보안 차단 안내를 확인한다. |

## 소스 빌드

앱 실행만 확인할 때는 소스 빌드가 필요하지 않다. 소스 검증이 필요한 경우 프로젝트 루트에서 다음 명령을 실행한다.

```bash
make check
make package
```

`make check`는 테스트와 Swift 패키지 빌드를 실행한다. `make package`는 `.build`에 Universal2 앱, ZIP, DMG와 체크섬 파일을 만든다.
