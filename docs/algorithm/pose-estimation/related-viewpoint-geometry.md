# 관련 연구 — 시점별 자세 기하

## 문서 요약

| 항목 | 내용 |
|---|---|
| 문서 유형 | 시점별 기하 방식 조사 |
| 적용 상태 | 시점별 판정 라우팅은 미채택, 공통 feature의 측면 ROI 기하는 채택 |
| 다루는 범위 | 측면 CVA, 정면 깊이축, 시점별 분기의 비용 |
| 제품 내 역할 | 시점별 자동 라우팅을 넣지 않고 같은 relative-depth feature의 입력 기하만 안정화하는 근거 기록 |

## 확인된 사실

- CVA는 C7과 귀 tragus를 기준으로 표준화된 측면 사진에서 측정하는 지표다.
- 측면 landmark 기반 FHP 분류 연구는 존재하지만, 촬영 자세·대상 집단·라벨이 현재 정면 Mac 웹캠 조건과 다르다.
- 정면 카메라에서는 머리의 전방 이동이 깊이축에 놓이므로 2D 각만으로 실제 전방 거리를 직접 복원할 수 없다.
- 단안 3D pose와 단안 depth는 이 모호성을 학습 prior로 추정하지만 센서 실측값은 아니다.

## 현재 플로우에 넣지 않는 이유

정면·측면·3/4 시점별로 서로 다른 feature를 운영하면 다음 항목이 각각 추가된다.

- 시점 분류와 전환 안정화
- 시점별 baseline과 임계
- 전환 중 `noEval` 정책
- 각 경로의 별도 데이터와 성능 검증

이 복잡도는 PoseNet·Vision 2D + DA-V2 Small이라는 현재 경로의 유효성을 먼저 확인하는 데 방해가 된다. 따라서 시점별 자동 라우팅, 측면 전용 각도 판정, Vision 3D fallback은 미채택이다. 다만 한쪽 귀와 머리 아래의 신뢰 어깨를 확인할 수 있는 측면·3/4 입력은 별도 판정 경로가 아니라 같은 relative-depth feature의 ROI 기하로 처리한다.

## 현재 적용

PoseNet 우선·Vision 2D fallback은 표준 어깨 기하 또는 제한된 머리 기준 측면 기하로 머리·몸통 ROI와 품질을 제공하고, DA-V2 Small은 relative depth를 제공한다. 프로젝트 자세 분석기가 같은 개인 baseline과 시간 조건으로 최종 상태를 결정한다. 머리 기준 측면 기하로도 정상 자세를 확인할 수 없는 큰 회전은 자세 기인 평가 불가로 기록하고, 그런 프레임이 버스트의 과반이면 비정상 증거로 처리한다. 사람이나 대상을 확인할 수 없는 시점 변화는 `noEval`로 처리하며 다른 판정 알고리즘으로 자동 전환하지 않는다.

## 재검토 조건

현재 경로가 충분한 제품 데이터에서도 자세를 분리하지 못하고, 측면 유도가 사용자 경험상 허용된다는 근거가 생긴 경우에만 별도 기획으로 재검토한다.
