# 관련 연구 — 시점별 자세 기하

## 문서 요약

| 항목 | 내용 |
|---|---|
| 문서 유형 | 시점별 기하 방식 조사 |
| 적용 상태 | 미채택, 리서치 참고 자료 |
| 다루는 범위 | 측면 CVA, 정면 깊이축, 시점별 분기의 비용 |
| 제품 내 역할 | 시점별 자동 라우팅을 현재 플로우에 넣지 않는 근거 기록 |

## 확인된 사실

- CVA는 C7과 귀 tragus를 기준으로 표준화된 측면 사진에서 측정하는 지표다.
- 측면 landmark 기반 FHP 분류 연구는 존재하지만, 촬영 자세·대상 집단·라벨이 현재 정면 Mac 웹캠 조건과 다르다.
- 정면 카메라에서는 머리의 전방 이동이 깊이축에 놓이므로 2D 각만으로 실제 전방 거리를 직접 복원할 수 없다.
- 단안 3D pose와 단안 depth는 이 모호성을 학습 prior로 추정하지만 센서 실측값은 아니다.

## 현재 플로우에 넣지 않는 이유

정면·측면·3/4 시점별로 서로 다른 feature를 운영하면 다음 항목이 각각 추가된다.

- 시점 분류와 전환 안정화
- 시점별 baseline과 임계
- 전환 중 `noEval` 정책
- 각 경로의 별도 데이터와 성능 검증

이 복잡도는 Vision 2D + DA-V2 Small이라는 현재 경로의 유효성을 먼저 확인하는 데 방해가 된다. 따라서 시점별 자동 라우팅, 측면 전용 각도 판정, Vision 3D fallback은 미채택이다.

## 현재 적용

고정된 정면 입력에서 Vision 2D는 머리·몸통 ROI와 품질을 제공하고, DA-V2 Small은 relative depth를 제공한다. 프로젝트 자세 분석기가 개인 baseline과 시간 조건으로 최종 상태를 결정한다. 고개 회전이나 큰 시점 변화로 비교가 어려우면 다른 알고리즘으로 전환하지 않고 `noEval`로 처리한다.

## 재검토 조건

현재 경로가 충분한 제품 데이터에서도 자세를 분리하지 못하고, 측면 유도가 사용자 경험상 허용된다는 근거가 생긴 경우에만 별도 기획으로 재검토한다.
