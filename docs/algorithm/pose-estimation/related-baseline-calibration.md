# 관련 연구 — 개인 baseline 보정

## 문서 요약

| 항목 | 내용 |
|---|---|
| 문서 유형 | baseline 방식 조사 |
| 적용 상태 | 안내된 개인 baseline 채택, 자동 적응은 미채택 |
| 다루는 범위 | baseline이 필요한 이유, 산출 원칙, 갱신 경계 |
| 제품 내 역할 | 개인 기준 사용과 자동 갱신 금지의 근거 정리 |

## 왜 개인 baseline인가

앱의 relative-depth feature는 절대 cm나 임상 CVA가 아니다. 사용자 체형, 카메라 위치, 의자와 화면 배치의 영향을 받으므로 사람 간 하나의 절대 임계보다 같은 사용자의 안내된 중립 자세 대비 변화를 우선한다.

CVA 문헌도 FHP 임계가 연구마다 다르고, 표준화된 측면 사진의 CVA조차 일반 웹캠 신호와 동일하지 않음을 보여 준다. 이것은 특정 baseline 알고리즘의 정확도를 증명하지는 않지만, 앱 내부 신호에 임상 절대 임계를 바로 옮기지 않아야 한다는 근거가 된다.

## 현재 방식

1. 사용자가 안내에 따라 중립 자세를 취한다.
2. landmark·ROI·depth 품질을 통과한 여러 프레임만 수집한다.
3. 중앙값을 중심값으로, MAD 또는 IQR을 변동 범위로 기록한다.
4. 일상 버스트의 대표값을 이 분포와 비교한다.
5. baseline을 다시 만들 때도 사용자의 명시적인 재보정 과정을 거친다.

최소 프레임 수, 허용 분산, 판정 임계는 리서치에서 임의로 가져오지 않고 제품 데이터로 정한다.

## 사용하지 않는 방식

- 일상 판정에서 `good`으로 나온 값을 baseline에 계속 누적하는 방식
- rolling percentile로 기준을 자동 이동하는 방식
- CUSUM 같은 drift detector가 baseline을 자동 갱신하는 방식

이 방식들은 일반 시계열 처리에는 쓰이지만 자세 도메인의 직접 근거가 없고, 반복된 나쁜 자세를 새로운 정상으로 흡수할 수 있다. 현재는 로직 복잡도를 늘리지 않고 재보정 안내만 사용한다.

## 검증할 것

- 중립 자세 반복 측정의 개인 내 분산
- 다른 날·다른 조명·다른 카메라 높이에서 baseline의 재현성
- baseline 대비 변화량과 자세 라벨의 분리도
- 재보정이 필요한 조건과 사용자 안내 방식
